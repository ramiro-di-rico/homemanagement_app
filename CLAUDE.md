## Architecture

This is a **Flutter personal finance app** supporting mobile (Android/iOS), desktop (Linux), and web. It manages accounts, transactions, budgets, recurring payments, and shared household access via invitations.

### Dependency Injection

The app uses **GetIt** as a service locator (no BLoC/Provider/Riverpod). All services and repositories are registered in `lib/main.dart` via `registerDependencies()` and `registerServices()` (generated by `injectable`). Widgets retrieve dependencies via `GetIt.instance<T>()`.

### Data Flow

```
HTTP Endpoint Service  →  Repository  →  Widget (via GetIt)
```

- **`lib/services/endpoints/`** — Raw HTTP calls to the backend API. One class per resource (e.g., `TransactionService`, `AccountService`).
- **`lib/services/repositories/`** — Business logic layer that wraps endpoint services, handles errors, and aggregates data.
- **`lib/views/`** — Stateful/Stateless widgets that call repositories directly. State is held in widgets via `setState`.

### Navigation

GoRouter is configured in `lib/routing.dart`. An authentication guard redirects unauthenticated users to `/login`. Routes under `/home` require authentication. The app supports deep links (handled by `DeepLinkingService`) including QR-scanned invite links.

Key route hierarchy:
- `/login`, `/registration` — auth flow (mobile and desktop variants)
- `/home` — authenticated shell with sub-routes: `/accounts/:id`, `/settings`, `/statistics`, `/budget`, `/transactions/search`, `/bulk-transactions`, `/invites`

### Platform Awareness

`PlatformContext` (in `lib/services/infra/`) detects mobile vs. desktop/web at runtime. Many views have separate mobile and desktop implementations (e.g., `LoginView` vs. `DesktopLoginView`). The routing layer selects the correct variant.

### Authentication & Security

`AuthenticationService` manages JWT tokens, biometric auth (via `local_auth` on mobile), and secure storage. `CryptographyService` provides AES encryption for sensitive local data.

### Key Infrastructure

- **`lib/services/infra/caching.dart`** — In-memory cache for API responses
- **`lib/services/infra/logger.dart`** — App-wide logging
- **`lib/services/infra/platform_context.dart`** — Platform detection
- **`lib/models/http-models/`** — Deserialization wrappers for API responses
- **`lib/models/view-models/`** — UI-specific data shapes
